This page documents the measures taken by the XRBitcoin and XRBitcoinCash projects to ensure complete clarity, security, and trustworthiness. It affirms that:

  • All public frontend code is open or verifiable; no hidden or off-chain code exists that alters behavior stealthily.
  • The AI features (chat interface, auditing, etc.) operate through a publicly known proxy and do not embed malicious logic or data harvesting.
  • Supply, issuer address, and transaction flows are auditable on the XRPL ledger; users may verify all claims independently via explorers (e.g. Bithomp).
  • Third-party dependencies (libraries, APIs) are vetted, minimal, and pinned where possible.
  • No private keys or secret code is stored in the frontend; interactions are forward through proxy to trusted APIs only.
  • All updates, audits, and security file revisions (security.txt, CSAF) are timestamped and publicly documented.

This statement is further supported by:

  • Auditability via XRPL explorers (e.g. Bithomp, XRPL Labs) for account and transaction transparency.
  • Open architecture with AI proxy endpoint logs subject to monitoring.
  • Community reviews, bug bounty programs (when instituted), and continuous integration of security reviews.

We encourage community members, external auditors, or curious users to raise any questions via the AI Control Center or file issues on the public GitHub repository. Our goal is to maintain a sacred space of trust in the XRPL environment â€” one built on clarity, not obfuscation.
